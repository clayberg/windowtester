What's New in 3.6.0

This document contains descriptions of some of the more interesting or significant changes in the WindowTester Pro v3.6.0 release.

  • User Interface
  • Core Runtime

User Interface

Recorder Console

The once experimental recorder console has been promoted! The new WindowTester Pro recording console displays recorded events giving users instant feedback during the recording process.

Using the console, users can:

  • Monitor and Edit Recorded Actions
  • Generate Multiple Tests during a Single Recording Session
  • Start and Stop the Recording Session
  • Add Assertion Hooks

More details can be found in the console reference.

Assertions UI

Using the new Assertions UI, assertions can be generated at recording time using the assertion inspector tool.



See this reference for more details.

Core Runtime

Condition Factories

A convenient approach to building conditions is to invoke locator-specific condition factory methods on a target locator.  For example,

   new ButtonLocator("Ok").isEnabled()  

produces a condition that tests for the enablement of a button labeled "Ok".   This release continues our evolution of this feature by adding a few new  factory methods:

  • TableCellLocator.hasText(..) - TableCellLocators can now be used to create HasTextConditions (for example: ui.assertThat(new TableCellLocator("item 1", "Value").hasText("yyy"))).
  • TreeItemLocator.isSelected() - TreeItemLocators now implement IsSelected and can now also be used to create IsSelectedConditions to test for the check-state of a tree item (for example: ui.assertThat(new TreeItemLocator("Popeye").isSelected()).

Locator Factories

Locator factories can be used to build readable tests and ease writing tests by hand. 

Tests written using factories read more fluently than their explicit constructor counterparts.  (Note: for maximum effect, factories are best imported statically.)  As an example, take: 

 ui.click(treeCell("foo/bar").at(column(3)).in(view("MyView"));
and notice that it reads much like a specification.

This approach to building locators is under active development.  This release includes the following factories:

  • SWTLocators -  creates common SWT locators.
  • EclipseLocators - creates common Eclipse locators.

Pluggable ScreenCapture

The global screen capture policy now user-configurable. In addition, the default screen capture handling implementation has been improved to be more robust in the face of out of memory conditions.

See the ScreenCapture javadocs for examples and details.
