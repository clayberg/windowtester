Introduction

This document describes setting up your development environment to use WindowTester Pro on your RCP project.

Project Setup

We assume that a custom target platform has been setup for the application under development.

Always use target platforms for your RCP development. Target platforms aggregate all of the bundles that are available to your application, cleanly separating the application under development from the environment in which it is being developed. Using target platforms enforces standardization at the level of the application (and not at the IDE). This standardization is key to managing dependencies and the build process.

Next, we expect that the target is managed using a target definition file.
