History


Version 3.5.1 (beta) -- September 5, 2008

Features:
  • TableCellLocator.hasText(..) - TableCellLocators can now be used to create HasTextConditions (for example: ui.assertThat(new TableCellLocator("item 1", "Value").hasText("yyy"))).
  • TreeItemLocator.isSelected() - TreeItemLocators now implement IsSelected and can now also be used to create IsSelectedConditions to test for the check-state of a tree item (for example: ui.assertThat(new TreeItemLocator("Popeye").isSelected()).
  • New WT Intro Page Content - New WindowTester Pro Intro content pages have been added making it easier to quickly navigate to useful resources from the Universal Intro Page.
  • Assertion UI Activated - The assertion generation UI that has been considered experimental until now, has been promoted to the main-stream and is now activated by default.
Fixes: 
  • TableCellLocator Row and Column by-text Handling - An issue preventing row and column text to be useable in specifiying table cell conditions (e.g., new TableCellLocator("item 1", "Value").isVisible()) has been fixed.
  • Double-Escape Handling in TreeItem Paths -TreeItem paths with double-escapes (e.g., "foo\\\\bar") where not being properly handled; this has been fixed..

Version 3.5.1 (beta) -- August 28, 2008

Fixes:
  • Drag And Drop Source Caching - drag sources that were established using moveTo(..) were not reliably getting cached. This has been fixed.
  • GEF Rendered Pane Handling - figures on rendered panes (such as those used by GMF) require (and now get) special bounds (re)translation.

Version 3.5.1 (beta) -- August 20, 2008

Fixes:
  • Figure Matcher Visibility Testing - The default figure matcher implementation now ensures figures are visible (rejecting figures that are not showing as indicated by the IFigure.isShowing() method).
  • Runtime DragTo Handling with Figures as Targets (revisited for XYs) - A bug in the runtime Drag and Drop support where the drag target is a figure and is wrapped in an XY locator has been addressed and fixed.

Version 3.5.1 (beta) -- August 3, 2008

Fixes:
  • ScreenCapture improvements - ScreenCapture handling has been improved to be more robust in the face of out of memory conditions. It is also now more user-configurable. See ScreenCapture for details.
  • RCP Ant Example Documentation Update - The documentation for the RCP Ant example has been updated to reflect refactoring in progress.
  • Runtime DragTo Handling with Figures as Targets - Runtime Drag and Drop support has been improved for the case where the drag target is a figure.

Version 3.5.1 (beta) -- July 17, 2008

Fixes:
  • Javadoc improvements - The docs for IUIContext.keyClick(int) were improved to better describe composite key stroke specifications.
  • Table Item Selection Recording - A regression was fixed that was causing double clicks on table items to be wrongly generated as single clicks.
  • Mac Native Library Linking - The Cocoa JNI library was wrongly specified. This has now been fixed.
  • WT.END Key Entry - A bug preventing proper dispatch of "END" keystrokes has been fixed.
  • XY Coordinates for Canvas Drops Fixed - XY Coordinates for drags to Canvas widgets were being (wrongly) discarded. This has now been fixed and recorded drags to canvases should include proper XY information.

Version 3.5.1 (beta) -- July 8, 2008

Fixes:
  • DragTo Modifier Keys - Modifier keys were being neglected by calls to dragTo; this has now been fixed.
  • Runtime handling of Menu Items w/ Slashes (and Shortcuts) - The MenuItem path tokenizer has been fixed to handle corner cases with slashes (e.g., "Find/Replace...") and accelerators/shortcuts.
  • JobsCompleteCondition Now Ignores EPP UDC Jobs - The JobsCompleteCondition has been updated to ignore EPP Usage Data Collector jobs by default.
  • Conditions Handled During Exception Handling - Registered conditions are now handled during shell tear down when for failed tests. This provides a hook for developers who need to participate in teardown. An example of where this is handy is in the case where a closed shell gives rise to a confirmation dialog. In order to not go into an infinite regress when the framework tries to close the shells, a condition can be registered to handle the nested dialog properly.
  • One Time Setup/Teardown - The logic to manage the test lifecycle has been improved and onetime class setup and teardown methods are now being called as expected.
  • ViewLocator.forName(..) - A new factory for View Locators has been implemented, allowing testers to specify views by name (vs. id).

Version 3.0.1 (beta) -- June 11, 2008

Features:
  • Palette Button Recording - Interactions with GEF palette buttons (e.g., the control used to hide/reveal the palette) now generate PaletteButtonLocator selections.
Fixes:
  • Dynamic Tree Content Handling Fix - Dynamic trees contain "dummy" items as placeholders for their dynamic content which were confusing our finder. These placeholders are now properly handled.
  • Test Execution Notification Fix - Timeouts in tests were bypassing our test finished event notification compromising some test lifecycle management (e.g., resetting of JFace Automation flags).
  • Palette Item Selection Fix/Workaround For Redesigned GEF Palette - In GEF 3.4RC3, a quirk in the newly redesigned palette model broke selections of the "Marquee" tool. This issue has now been addressed in our EditPartFinder.

Version 3.0.1 (beta) -- May 29, 2008

Features:
  • Recorder Console Enabled by Default - The recorder console has made it out of its experimental period and is now on by default. The "classic" recorder can be re-enabled from the WindowTester Pro Recorder UI preferences. For more details, see the online help docs.
Fixes:
  • GEF Runtime Transaction Fix - Transactions triggered by actions on a GEF canvas are waited for. In cases where a modal dialog was popped up before the transaction was complete, this wait was timing out (pending interaction with the dialog). This case is now handled (the wait is canceled).
  • JUnit38Runner Fixed - The provisional JUnit38 runner provided for JUnit4 compatibility had a faulty filter implementation that has been fixed.

Version 3.0.1 (beta) -- May 20, 2008

Fixes:
  • Empty Shell Showing Fixes - Transient shells that were causing spurious wait calls to be generated are now being identified and ignored by the recorder.
  • Debugging Warnings Changed to Traces - When the recorder cannot find the control with current focus it used to add a remark to the log, this is now a trace (and will only be seen if tracing is enabled).
  • Recorder DND Glitches Fixed - A number of issues in GEF DND recording have been addressed: a bug that caused empty dragTo(..) events to be codegened has been fixed; and drag events whose source selection is missing from the recording (because there was no explicit click before the drag) are being repaired at codegen-time.
  • Ant Example Documentation Updated - The ant example docs were updated to include a tip about properly setting up proxy host information. (Thanks to Fabian Leuthold for pointing this out!)
Features:
  • CTabItem Close Runtime support - CTabItems can now be closed by passing a CTabItemLocator into the ui.close(..) method. Closing this way rather than by using a relative XY to locate the 'X' on the tab is not only more robust but also better communicates intent in the test.
  • CTabItem Close Recorder support - CTabItem interactions no longer include XY information. In case the CTabItem was closed, a call to ui.close(..) will be generated.

Version 3.0.1 (beta) -- April 30, 2008

Fixes:
  • GEF Scroll and Reveal Fixes - Fixes were made to the GEF runtime to better handle scrolling and revealing items on a figure canvas during selection.
  • GEF Palette Item Revealing Strategy - The strategy used to reveal palette items when the palette is hidden has been updated to pin the palette. This fix addresses potential timing issues where clicks on items occur as the palette is opening or closing.
  • TreeItemLocator Empty Child Reveal Fixes - Handling of the case where a tree item selection is attempted on a branch that does not exist has been improved. (The new behavior is to throw a search exception where before there was an expansion timeout exception.)
  • Recorder Console Swing Improvements - The Recorder console now has improved labels for Swing element selections.
  • Filtered Tree Item Recording fixes - The recorder has been fixed to recognize filtered trees.

Version 2.3.1 (beta) -- released December 21, 2007

Features:
  • Added support to change the keyboard type to current locale during the test.
For example, If current locale is German / Germany, and the test needs to use the German Keyboard to enter text, then in the test have the statement
    
WT.setWTLocaleToCurrent();

From then on, all text will be entered using the German keyboard mapping. At the end of the tests, in order to set it back to the US Keyboard use

WT.resetWTLocale();

  • New Recording Information dialog -- Added a new dialog that appears after a recording session if nothing was recorded so that the cause of the recording problem can be better deduced.
  • Split local and global ConditionMonitor -- Added a condition monitor associated with the IUIContext separate from the global condition monitor. The local condition monitor can be accessed via the IUIContext#getAdapter(...) method by passing IConditionMonitor.class as the argument. The local condition monitor and all conditions and handlers associated with it are discarded when the IUIContext is discarded (typically at the end of the execution of a test), while the global condition monitor persists for the entire java execution session.
  • Split local and global ShellMonitor -- Added a shell monitor associated with the IUIContext separate from the global shell monitor. The local shell monitor can be accessed via the IUIContext#getAdapter(...) method by passing IShellMonitor.class as the argument.The local shell monitor and all conditions and handlers associated with it are discarded when the IUIContext is discarded (typically at the end of the execution of a test), while the global shell monitor persists for the entire java execution session.

Fixes:
  • Test Failure Teardown --- some test failures (where menus or modal dialogs were open at the time of failure) were blocking test execution; this has now been fixed and tests fail appropriately.
  • Context Menu Support --- context menu support has been improved in runtime and playback.
  • Escaped Text in Text Entry Recordings --- backslashes recorded in text entry are now properly escaped in the generated text entry code.
  • Support for Slashes ('\') in Path Items --- Path items that use the default delimiter can now be handled as the slash is escaped. For example: the item "Find/Replace" becomes "Find\\/Replace".
